# 命令行工具使用指南 -- 本文档详细介绍了 ShitBot 工具模块中命令行相关工具的使用方法、参数说明、使用示例以及最佳实践。

## 工具概述

命令行工具模块包含一个核心工具：`shell_command`（执行 shell 命令）。该工具允许用户在系统上执行各种命令行指令，完成系统操作、文件管理、程序运行等任务。

## shell_command - 执行 shell 命令

### 工具介绍

`shell_command` 工具用于在用户的操作系统上执行 shell 命令。在 Windows 系统上，可以执行 CMD 命令；在 Linux/Mac 系统上，可以执行 Bash 命令。这是一个非常强大的工具，可以完成各种系统级操作。

### 工具参数

| 参数名 | 类型 | 必需 | 描述 |
|--------|------|------|------|
| command | string | 是 | 要执行的 shell 命令 |

### 使用示例

#### 示例 1：列出当前目录内容
```json
{
  "command": "dir"
}
```

#### 示例 2：创建目录
```json
{
  "command": "mkdir new_folder"
}
```

#### 示例 3：查看系统信息
```json
{
  "command": "systeminfo"
}
```

#### 示例 4：运行 Python 脚本
```json
{
  "command": "python script.py"
}
```

#### 示例 5：查看进程列表
```json
{
  "command": "tasklist"
}
```

## 常见问题

### Q1：如何执行包含空格的命令？
A：使用引号包裹包含空格的路径或参数。例如：
```json
{
  "command": "dir \"C:\\Program Files\""
}
```

### Q2：如何执行多条命令？
A：在 Windows 上使用 `&` 或 `&&` 连接命令，在 Linux/Mac 上使用 `;` 或 `&&` 连接命令。例如：
```json
{
  "command": "cd C:\\ && dir"
}
```

### Q3：命令执行失败怎么办？
A：检查命令语法是否正确，确保命令在当前系统上可用，检查是否有足够的权限执行该命令。

## 注意事项

1. **安全第一**：执行命令前请确认命令的安全性，避免执行可能造成系统损坏的命令。
2. **权限问题**：某些命令需要管理员权限才能执行。
3. **系统差异**：不同操作系统的命令语法可能不同，请根据当前系统选择合适的命令。
4. **路径格式**：Windows 路径使用反斜杠 `\`，Linux/Mac 路径使用正斜杠 `/`。
5. **命令输出**：命令的输出结果会返回给调用者，请注意输出内容的长度。

## 最佳实践

1. **测试命令**：在执行重要命令前，先在本地测试命令的正确性。
2. **使用绝对路径**：对于文件操作，建议使用绝对路径以避免路径错误。
3. **错误处理**：考虑命令可能失败的情况，做好错误处理。
4. **日志记录**：对于重要的命令执行，建议记录日志以便追踪。
5. **权限控制**：避免执行需要过高权限的命令，除非确实必要。
