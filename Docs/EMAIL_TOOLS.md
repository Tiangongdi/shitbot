# 邮件工具使用指南 -- 本文档详细介绍了 ShitBot 工具模块中邮件相关工具的使用方法、参数说明、使用示例以及最佳实践。

## 工具概述

邮件工具模块包含一个核心工具：`send_email`（发送邮件）。该工具用于向指定邮箱地址发送邮件，支持自定义主题和正文内容。

## send_email - 发送邮件

### 工具介绍

`send_email` 工具用于向指定的邮箱地址发送邮件。可以设置邮件的主题和正文内容，适用于发送通知、报告、提醒等各类邮件。

### 工具参数

| 参数名 | 类型 | 必需 | 描述 |
|--------|------|------|------|
| to_email | string | 是 | 收件人邮箱地址 |
| subject | string | 是 | 邮件主题 |
| body | string | 是 | 邮件正文内容 |

### 使用示例

**示例 1：发送简单通知邮件**
```
send_email(
  to_email: "user@example.com",
  subject: "任务完成通知",
  body: "您的任务已成功完成，请查看详情。"
)
```

**示例 2：发送详细报告邮件**
```
send_email(
  to_email: "manager@company.com",
  subject: "周报 - 2026年第6周",
  body: "本周工作总结：\n1. 完成了项目A的开发\n2. 修复了3个bug\n3. 参加了2次团队会议\n\n下周计划：\n1. 开始项目B的开发\n2. 优化系统性能"
)
```

### 最佳实践

1. **邮件主题要清晰**：使用简洁明了的主题，让收件人快速了解邮件内容
2. **正文格式化**：使用换行符和适当的格式让邮件更易读
3. **验证邮箱地址**：确保收件人邮箱地址正确，避免发送失败
4. **内容简洁**：邮件正文不宜过长，重要信息放在前面
5. **敏感信息保护**：不要在邮件中发送密码、密钥等敏感信息

## 典型使用场景

1. **任务通知**：任务完成后自动发送通知邮件
2. **报告发送**：定期发送工作报告、数据报告
3. **提醒服务**：定时发送提醒邮件
4. **系统告警**：系统出现问题时发送告警邮件

## 注意事项

1. 确保邮件服务器配置正确
2. 避免频繁发送大量邮件，可能被识别为垃圾邮件
3. 邮件内容要符合法律法规要求
4. 注意保护用户隐私，不要泄露他人邮箱地址

## 快速参考

```
send_email(to_email, subject, body)
```
