# 输入工具使用指南 -- 本文档详细介绍了 ShitBot 工具模块中输入相关工具的使用方法、参数说明、使用示例以及最佳实践。

## 工具概述

输入工具模块包含两个核心工具：`input`（用户输入）和 `input_y_or_n`（用户确认）。这两个工具用于与用户进行交互，获取用户的输入或确认。

## input - 用户输入

### 工具介绍
`input` 工具用于向用户提出输入提示，并获取用户的输入内容。适用于需要用户自由输入文本的场景。

### 工具参数
- `prompt`（必需）：向用户提出的输入提示，例如"请输入您的姓名"

### 使用示例
```
用户：帮我创建一个文件
助手：好的，请告诉我文件名是什么？
（调用 input 工具，prompt="请输入文件名"）
用户：test.txt
助手：好的，正在创建 test.txt 文件...
```

### 最佳实践
- 提示语要清晰明确，让用户知道需要输入什么
- 避免使用过于复杂的提示语
- 在需要用户输入关键信息时使用此工具

## input_y_or_n - 用户确认

### 工具介绍
`input_y_or_n` 工具用于向用户提出确认提示，用户只能回答"是"或"否"。适用于需要用户确认操作的场景。

### 工具参数
- `prompt`（必需）：向用户提出的确认提示，例如"您确定要删除这个文件吗？"

### 使用示例
```
用户：删除 test.txt 文件
助手：您确定要删除 test.txt 文件吗？此操作不可恢复。
（调用 input_y_or_n 工具，prompt="您确定要删除 test.txt 文件吗？"）
用户：是
助手：好的，正在删除 test.txt 文件...
```

### 最佳实践
- 在执行不可逆操作前使用此工具进行确认
- 提示语要明确说明操作的后果
- 让用户清楚知道确认后的影响

## 典型使用场景

### 场景1：创建文件时询问文件名
```
用户：创建一个新文件
助手：好的，请输入文件名：
（调用 input 工具）
用户：mydocument.txt
助手：正在创建 mydocument.txt 文件...
```

### 场景2：删除文件前确认
```
用户：删除 important.txt
助手：您确定要删除 important.txt 吗？删除后无法恢复。
（调用 input_y_or_n 工具）
用户：否
助手：好的，已取消删除操作。
```

### 场景3：询问用户偏好
```
用户：帮我搜索一些信息
助手：好的，请告诉我您想搜索什么内容？
（调用 input 工具）
用户：人工智能最新进展
助手：正在为您搜索人工智能最新进展...
```

## 常见问题

### Q: input 和 input_y_or_n 有什么区别？
A: `input` 允许用户自由输入任何文本内容，而 `input_y_or_n` 只能回答"是"或"否"，用于确认操作。

### Q: 什么时候应该使用 input_y_or_n？
A: 在执行可能造成数据丢失或其他不可逆操作之前，应该使用 `input_y_or_n` 让用户确认。

### Q: 提示语应该怎么写？
A: 提示语要简洁明了，让用户清楚知道需要做什么。对于 `input_y_or_n`，最好说明操作的后果。

## 注意事项

1. 不要滥用输入工具，避免频繁打断用户
2. 提示语要友好、礼貌
3. 在关键操作前务必使用 `input_y_or_n` 进行确认
4. 获取用户输入后要及时反馈，让用户知道操作正在进行
