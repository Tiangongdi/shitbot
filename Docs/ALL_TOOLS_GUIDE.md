# ShitBot 完整工具使用指南 -- 该文档详细介绍了 ShitBot 中所有可用工具的使用方法、参数说明以及最佳实践

## 工具概述

ShitBot 提供了丰富的工具集，涵盖网络搜索、文件操作、定时任务、邮件发送、记忆管理等多个功能模块。本文档将逐一介绍每个工具的使用方法。

---

## 网络搜索模块

### search_web - 网络搜索

#### 工具介绍
在网络上搜索信息，支持自定义搜索关键词和返回结果数量。

#### 工具参数
- `query` (必需): 搜索查询词，字符串类型
- `count` (可选): 返回结果数量，整数类型，默认值为 5

#### 使用示例
```
搜索最新的AI新闻，返回10条结果
参数：query="AI最新新闻", count=10
```

#### 最佳实践
- 使用具体的关键词可以获得更精准的搜索结果
- 如果需要更多结果，可以适当增加 count 参数值
- 建议配合 browse_page 工具使用，获取详细内容

---

### browse_page - 网页浏览

#### 工具介绍
打开指定的网页并提取内容，支持指定提取目标。

#### 工具参数
- `url` (必需): 要访问的网页 URL，字符串类型
- `query` (必需): 告诉网站提取总结助手要在这个网站里面提取和总结的内容，字符串类型

#### 使用示例
```
访问百度首页并提取搜索功能相关信息
参数：url="https://www.baidu.com", query="搜索功能"
```

#### 最佳实践
- query 参数要明确指定需要提取的内容类型
- 建议先使用 search_web 获取相关网页链接，再使用 browse_page 获取详细内容
- 对于大型网站，query 越具体，提取效果越好

## 文件操作模块

### write_file - 写入文件

#### 工具介绍
给指定文件写入内容，如果文件不存在会自动创建。

#### 工具参数
- `file_path` (必需): 要写入的文件路径，字符串类型
- `content` (必需): 要写入的内容，字符串类型

#### 使用示例
```
在 D 盘创建一个文本文件并写入内容
参数：file_path="D:\\test.txt", content="Hello World"
```

#### 最佳实践
- Windows 路径需要使用双反斜杠 `\\` 或正斜杠 `/`
- 写入前确认路径存在，否则可能失败
- 大文件写入建议分批进行

---

### read_file - 读取文件

#### 工具介绍
读取指定文件的内容。

#### 工具参数
- `file_path` (必需): 要读取的文件路径，字符串类型

#### 使用示例
```
读取 D 盘下的配置文件
参数：file_path="D:\\config.json"
```

#### 最佳实践
- 确保文件路径正确且文件存在
- 大文件读取可能需要较长时间
- 注意文件编码格式

---

### copy_file - 复制文件

#### 工具介绍
复制指定文件到目标路径。

#### 工具参数
- `source_path` (必需): 要复制的文件路径，字符串类型
- `dest_path` (必需): 要复制到的目标路径，字符串类型

#### 使用示例
```
将文件从源目录复制到目标目录
参数：source_path="D:\\source\\file.txt", dest_path="D:\\backup\\file.txt"
```

#### 最佳实践
- 确保目标目录存在
- 如果目标文件已存在，会被覆盖
- 复制大文件时请耐心等待

---

### move_file - 移动文件

#### 工具介绍
移动指定文件到目标路径。

#### 工具参数
- `source_path` (必需): 要移动的文件路径，字符串类型
- `dest_path` (必需): 要移动到的目标路径，字符串类型

#### 使用示例
```
将文件移动到归档目录
参数：source_path="D:\\temp\\file.txt", dest_path="D:\\archive\\file.txt"
```

#### 最佳实践
- 移动操作会删除原位置的文件
- 确保目标目录存在
- 跨磁盘移动可能需要复制+删除两步操作

---

### create_dir - 创建目录

#### 工具介绍
创建指定路径的目录。

#### 工具参数
- `dir_path` (必需): 要创建的目录路径，字符串类型

#### 使用示例
```
创建项目目录
参数：dir_path="D:\\projects\\myproject"
```

#### 最佳实践

- 如果目录已存在，不会报错
- Windows 路径使用双反斜杠或正斜杠

---

### get_dir_content - 获取目录内容

#### 工具介绍
获取指定目录下的文件和子目录列表。

#### 工具参数
- `dir_path` (必需): 要获取内容的目录路径，字符串类型

#### 使用示例
```
查看项目目录下的所有文件
参数：dir_path="D:\\projects"
```

#### 最佳实践
- 可以用于检查目录结构
- 配合其他文件操作工具使用
- 空目录会返回空列表

---

### delete_file - 删除文件

#### 工具介绍
删除指定的文件。

#### 工具参数
- `file_path` (必需): 要删除的文件路径，字符串类型

#### 使用示例

删除临时文件

## 邮件发送模块

### send_email - 发送邮件

#### 工具介绍
发送邮件到指定邮箱地址。

#### 工具参数
- `to_email` (必需): 收件人邮箱地址，字符串类型

## 定时任务模块

### once_after - 延迟执行一次

#### 工具介绍
在指定时间后执行一次任务。

#### 工具参数
- `time` (必需): 延迟时间（秒），整数类型
- `task` (必需): 任务描述，字符串类型

#### 使用示例
```
5分钟后提醒喝水
参数：time=300, task="提醒喝水"
```

#### 最佳实践
- 时间单位为秒
- 适用于一次性提醒任务
- 任务描述要清晰明确

---

### interval - 周期性执行

#### 工具介绍
按照指定时间间隔周期性执行任务。

#### 工具参数
- `interval_seconds` (必需): 时间间隔（秒），整数类型
- `interval_count` (必需): 触发次数，-1 表示无限次，整数类型
- `task` (必需): 任务描述，字符串类型

#### 使用示例
```
每小时检查一次系统状态，执行24次
参数：interval_seconds=3600, interval_count=24, task="检查系统状态"
```

#### 最佳实践
- interval_count=-1 表示无限循环
- 注意任务执行时间不要超过间隔时间
- 长期运行的任务要考虑资源占用

---

### daily_at - 每日定时执行

#### 工具介绍
每天在指定时间执行任务。

#### 工具参数
- `hour` (必需): 小时（0-23），整数类型
- `minute` (可选): 分钟（0-59），整数类型
- `task` (必需): 任务描述，字符串类型

#### 使用示例
```
每天早上8点发送日报
参数：hour=8, minute=0, task="发送日报"
```

#### 最佳实践
- 使用24小时制
- 时间格式要准确
- 适合日常固定任务

---

### cancel - 取消定时任务

#### 工具介绍
取消指定的定时任务。

#### 工具参数
- `task_id` (必需): 任务ID，字符串类型

#### 使用示例
```
取消ID为 task_123 的任务
参数：task_id="task_123"
```

#### 最佳实践
- 可以使用 list 工具查看所有任务ID
- 取消后任务将不再执行
- 已执行的任务无法取消

---

### pause - 暂停定时任务

#### 工具介绍
暂停指定的定时任务。

#### 工具参数
- `task_id` (必需): 任务ID，字符串类型

#### 使用示例
```
暂停ID为 task_456 的任务
参数：task_id="task_456"
```

#### 最佳实践
- 暂停后可以恢复执行
- 暂停期间任务不会触发
- 使用 resume 工具恢复任务

---

### resume - 恢复定时任务

#### 工具介绍
恢复已暂停的定时任务。

#### 工具参数
- `task_id` (必需): 任务ID，字符串类型

#### 使用示例
```
恢复ID为 task_789 的任务
参数：task_id="task_789"
```

#### 最佳实践
- 只能恢复已暂停的任务
- 恢复后任务按原计划继续执行
- 配合 pause 工具使用

---

### list - 列出所有定时任务

#### 工具介绍
列出当前所有的定时任务及其状态。

#### 工具参数
无

#### 使用示例
```
查看所有定时任务
参数：无
```

#### 最佳实践
- 定期检查任务状态
- 获取任务ID用于其他操作
- 可以查看任务是否正常运行

## 注意事项

1. **路径格式**: Windows 系统路径使用双反斜杠 `\\` 或正斜杠 `/`
2. **参数类型**: 注意参数的数据类型，字符串需要加引号
3. **错误处理**: 工具执行失败时会返回错误信息，请根据提示调整
4. **安全性**: 执行删除、移动等操作前请确认路径正确
5. **性能**: 大文件操作和复杂命令可能需要较长时间，请耐心等待
