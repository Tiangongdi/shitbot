# 文件操作工具使用指南 -- 本文档详细介绍了 ShitBot 工具模块中文件操作相关工具的使用方法、参数说明、使用示例以及最佳实践。

## 工具概述

文件操作模块提供了对本地文件系统的基本操作能力，包括文件的读取、写入、复制、移动、删除，以及目录的创建和查看等功能。这些工具可以帮助用户完成各种文件管理任务。

## write_file - 写入文件

### 工具介绍
`write_file` 工具用于向指定文件路径写入内容。如果文件不存在，会自动创建新文件；如果文件已存在，会覆盖原有内容。

### 工具参数
- `file_path` (必需): 要写入的文件路径，字符串类型
- `content` (必需): 要写入的内容，字符串类型

### 使用示例
```
写入文件到 d:\test.txt，内容为 "Hello World"
```

### 最佳实践
- 写入文件前建议先确认目标路径是否可访问
- 对于重要文件，建议先备份再写入
- 写入大文件时注意内容长度限制

---

## read_file - 读取文件

### 工具介绍
`read_file` 工具用于读取指定文件的内容并返回。

### 工具参数
- `file_path` (必需): 要读取的文件路径，字符串类型

### 使用示例
```
读取 d:\test.txt 文件的内容
```

### 最佳实践
- 读取前确认文件存在且可访问
- 对于大文件，注意返回内容的长度
- 处理文件编码问题，确保正确读取中文等内容

---

## copy_file - 复制文件

### 工具介绍
`copy_file` 工具用于将源文件复制到目标路径，保留原文件不变。

### 工具参数
- `source_path` (必需): 要复制的源文件路径，字符串类型
- `dest_path` (必需): 要复制到的目标文件路径，字符串类型

### 使用示例
```
将 d:\source.txt 复制到 d:\backup\source.txt
```

### 最佳实践
- 确保目标目录存在，否则可能复制失败
- 如果目标文件已存在，会被覆盖
- 复制大文件时注意磁盘空间

---

## move_file - 移动文件

### 工具介绍
`move_file` 工具用于将文件从源路径移动到目标路径，移动后原文件将被删除。

### 工具参数
- `source_path` (必需): 要移动的源文件路径，字符串类型
- `dest_path` (必需): 要移动到的目标文件路径，字符串类型

### 使用示例
```
将 d:\temp\file.txt 移动到 d:\documents\file.txt
```

### 最佳实践
- 移动操作不可逆，建议先备份重要文件
- 确保目标目录存在
- 移动文件会改变文件位置，但保留文件内容

---

## delete_file - 删除文件

### 工具介绍
`delete_file` 工具用于删除指定文件，删除后文件将无法恢复。

### 工具参数
- `file_path` (必需): 要删除的文件路径，字符串类型

### 使用示例
```
删除 d:\temp\old_file.txt
```

### 最佳实践
- 删除操作不可逆，请谨慎使用
- 删除前建议确认文件不再需要
- 对于重要文件，建议先备份再删除

---

## create_dir - 创建目录

### 工具介绍
`create_dir` 工具用于在指定路径创建新目录。

### 工具参数
- `dir_path` (必需): 要创建的目录路径，字符串类型

### 使用示例
```
在 d:\project 下创建新目录 new_folder
```

### 最佳实践
- 如果父目录不存在，可能需要先创建父目录
- 目录名称建议使用有意义的名称
- 避免使用特殊字符和空格

---

## get_dir_content - 获取目录内容

### 工具介绍
`get_dir_content` 工具用于列出指定目录下的所有文件和子目录。

### 工具参数
- `dir_path` (必需): 要获取内容的目录路径，字符串类型

### 使用示例
```
获取 d:\project 目录下的所有内容
```

### 最佳实践
- 确保目录存在且可访问
- 对于包含大量文件的目录，返回结果可能较多
- 可以用于检查目录结构或查找特定文件

---

## 典型使用场景

### 场景1：批量处理文件
1. 使用 `get_dir_content` 获取目录下的所有文件
2. 使用 `read_file` 读取每个文件内容
3. 处理内容后使用 `write_file` 写入新文件

### 场景2：文件备份
1. 使用 `copy_file` 将重要文件复制到备份目录
2. 定期执行备份操作确保数据安全

### 场景3：项目整理
1. 使用 `create_dir` 创建新的目录结构
2. 使用 `move_file` 将文件移动到对应目录
3. 使用 `delete_file` 清理不需要的文件

## 常见问题

**Q: 写入文件时提示权限不足怎么办？**
A: 请检查目标路径的写入权限，确保有足够的权限访问该位置。

**Q: 如何判断文件是否存在？**
A: 可以尝试读取文件，如果失败则文件可能不存在。

**Q: 移动文件和复制文件有什么区别？**
A: 复制会保留原文件，移动会删除原文件。

## 注意事项

1. 文件路径建议使用绝对路径，避免相对路径带来的混淆
2. 在 Windows 系统中，路径分隔符使用反斜杠 `\`
3. 操作重要文件前务必备份
4. 注意文件路径中的特殊字符转义
5. 大文件操作可能需要较长时间，请耐心等待
