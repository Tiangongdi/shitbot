# 技术选型报告生成器

## 功能概述

本技能整合多个维度的信息，生成全面的技术选型报告：
- **项目灵感** - 来自 idea-expander 的分支方案
- **市场调研** - 来自 market-research 的市场分析
- **开发者画像** - 来自 user-profile-generator 的开发者特征 
- **网络搜索** - 获取最新技术趋势和对比信息
- **历史记忆** - 参考过往的技术决策和经验

## 工作流程

### 第一步：收集前置信息

确认以下信息是否已准备好：
1. **项目灵感分支** - 用户选择的idea-expander分支
2. **市场调研报告** - market-research的分析结果
3. **开发者画像** - user-profile-generator生成的开发者特征

如果缺少任何信息，提示用户先生成相应内容。

### 第二步：获取历史记忆

根据前面的历史记忆总结下面的信息：     
- 这个项目是什么？
- 这个项目的目标是什么？
- 这个项目的功能需求有哪些？
- 这个项目的非功能需求有哪些？
- 这个项目的约束条件有哪些？
- 这个项目的开发者有什么特点？

### 第三步：网络搜索技术信息

使用 `search_web` 工具，搜索以下内容：

1. **技术栈对比**
   - 搜索关键词："[技术A] vs [技术B] 对比 优缺点"
   - 目的：了解不同技术方案的优劣

2. **技术趋势**
   - 搜索关键词："[技术领域] 2024/2025 技术趋势"
   - 目的：了解技术发展方向

3. **最佳实践**
   - 搜索关键词："[项目类型] 技术架构 最佳实践"
   - 目的：学习行业成熟方案

4. **性能对比**
   - 搜索关键词："[技术A] [技术B] 性能对比 benchmark"
   - 目的：获取客观数据支持

### 第四步：深度阅读关键资料

使用 `browse_page` 工具，深入阅读重要的技术文章和对比报告。

### 第五步：生成技术选型报告

整合所有信息，生成结构化的技术选型报告。

## 注意事项

1. **信息完整性** - 确保三个前置skill的信息都已收集
2. **搜索针对性** - 根据项目特点搜索相关技术信息
3. **客观公正** - 技术对比要基于事实，避免偏见或主观判断
4. **用户导向** - 结合用户画像推荐最适合的方案
5. **时效性** - 优先使用最新的技术信息和趋势
6. **历史参考** - 充分利用历史记忆中的经验教训
7. **使用的技术栈** - 使用的技术栈应该符合项目需求和团队，并且是当下企业开发最流行的技术栈，如果项目需要前端页面，建议参考ui-ux-pro-max技能，地址：.\Skills\ui-ux-pro-max\SKILL.md。
