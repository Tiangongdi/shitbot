# 项目灵感扩展工具

## 功能概述

本技能帮助用户将简短的项目灵感扩展成更完整的项目方向。通过分析用户提供的灵感核心，生成3个不同方向的分支，每个分支都有简短的介绍，让用户可以选择最适合自己的方向。

## 工作流程

### 第一步：收集灵感

向用户询问他们的项目灵感：

```
请告诉我你的项目灵感，越简短越好！我会帮你扩展成3个不同的方向供你选择。
```

### 第二步：生成三个分支

分析用户的灵感，生成3个不同方向的分支。每个分支包含：
- **分支名称** - 简洁的方向名称
- **简短介绍** - 2-3句话描述这个方向的特点和核心内容

**分支差异化维度：**
- 技术路线差异
- 用户群体差异
- 商业模式差异
- 规模定位差异
- 平台差异

### 第三步：展示分支方案，引导用户选择  

使用以下格式展示：

```
## 分支一：[分支名称]
[简短介绍，2-3句话]

## 分支二：[分支名称]
[简短介绍，2-3句话]

## 分支三：[分支名称]
[简短介绍，2-3句话]
```

展示完三个分支后，询问用户：

```
请选择你最喜欢的分支（1/2/3），或者告诉我你希望调整的方向！
```

## 注意事项

1. **保持核心价值** - 三个分支都应围绕用户原始灵感的核心价值展开
2. **差异化明显** - 三个分支之间应该有清晰的区别
3. **简洁明了** - 每个分支的介绍控制在2-3句话
4. **可行性** - 每个分支都应该是可实施的
