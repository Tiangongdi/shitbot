# 更新ShitBot项目代码流程
该流程是为了更好的规范用户需要你更新ShitBot项目代码的流程，确保项目代码的更新是安全、规范的，建议阅读Coder Role 来了解编码规范。  
## 工作文件目录
项目的文件是在"./Skills/project-update/project/"目录下，包括：
1. "LOG.md": 项目更新日志，记录每次更新的内容、时间、用户要求等信息。
2. "MUNO.md": 项目目录，介绍项目里面所有文件的作用、目录结构等信息。    
## 第一步
阅读上面的文件，了解项目的目录结构、文件作用等信息，如果没有，那么你就自己收集创建上面的文件，对这个项目有个充分的了解。
## 第二步
所有git命令缓存项目代码，避免后面代码写错崩溃，包括：
1. git add .
2. git commit -m "缓存历史记录"
## 第三步
阅读Coder Role 来了解编码规范，按照要求更新具体的功能，每一次更新代码都需要向用户进行一次汇报，包括：
1. 更新的功能
2. 更新的时间
## 第四步
所有更新完成，进行对两个工作文件的更新，包括：
1. "LOG.md": 项目更新日志，记录每次更新的内容、时间、用户要求等信息。
2. "MUNO.md": 项目目录，介绍项目里面所有文件的作用、目录结构等信息。        
## 第五步
git缓存这个项目，包括：
1. git add .
2. git commit -m "工作简介"
## 第六步
询问用户还有没有其他更新操作，如果没有就叫用户重启更新ShitBot
