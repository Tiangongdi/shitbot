# 初始化智能体

## 概述

本skill主要用于初始化智能体，配置智能体初始状态、设置智能体基础参数时进行提问用户收集必要信息，并更新 Self.txt 文件。

## 提问流程

首先询问用户是否确定要初始化智能体配置。如果用户确认，则按以下顺序依次提问：

1. **智能体名称**：请告诉我你希望智能体的名称是什么？
2. **智能体人格**：请描述一下智能体的人格特点或性格（例如：友好、专业、幽默等）？
3. **用户名称**：请问应该如何称呼你？
4. **用户介绍**：请介绍一下你自己（可选，用于让智能体更好地了解用户）？

收集完所有信息后，按照以下格式更新 `.\.shitbot\Self.txt` 文件：

```
# ShitBot Self-Information File

## Name: [智能体名称]
## Personality: [智能体人格]
## User: [用户名称]
## User Introduction: [用户介绍]
## Emoji: [智能体的emoji]
```
然后修改USER.md文件，添加用户信息。
更新完成后，告知用户初始化已完成。

## 如果当前是用户第一次使用，那么请帮忙配置项目参数 （Self里面有被*包裹的说明文字就是该用户第一次配置）
地址：.\.shitbot\data\settings.json
示例
```json
{
    "default_workflow": "sole"  , // 默认工作流，可选值：sole、coder、plan
    "max_conversation_count": 50, // 最大对话次数（超过次数后，智能体会保存记忆和token使用记录）
    "token_saving_mode": false, // 是否开启token节约模式
    "created_at": "2026-03-21" // 创建时间
}
```
阅读工作流文档，然后介绍提问用户是需要什么工作流作为默认工作流（sole、coder、plan）。
然后提问用户的具体需求，例如：
- 最大对话次数（超过次数后，智能体会保存记忆和token使用记录）
- 是否开启token节约模式（开启后，智能体会在对话中自动管理token，避免消耗token） 
最后更新settings.json文件。
[工作流对比文档](docs/workflows_comparison.md)
## 注意事项

- 如果用户不想填写某个信息，可以使用默认值或"无"
- 用户介绍为可选字段，可以留空
- 更新文件后删除此说明文字
