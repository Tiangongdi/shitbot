# 用户需求符合最大token使用数和是否开启token节约模式的要求介绍文档     
如果正在初始化智能体，那么请询问用户的具体需求，配置最大token使用数和是否开启token节约模式。
## 是否开启token节约模式
- **定义**: 是否开启token节约模式，开启后，系统会在对话中自动管理token，避免消耗token。
- **默认值**: false     
### 开启的要求
1. 用户操作是长期，并且用户并不希望自己经常通过"/new"指令来重新开始对话。"/new"命令是会保存当前的对话记录，总结一个记忆总结文档(比对话记录消耗更少的token)给智能体作为上下文，避免智能体忘记之前的干了什么，机制和开启token节约模式相同，但是是手动的。
2. 如果用户行为操作是短期临时任务，比如查询新闻、查询天气等，那么建议开启。
3. 如果用户行为操作是需要长期任务但是对上下文的要求不需要特别高，比如进行开发一个项目，那么建议开启。
4. 用户希望优化成本，建议开启。
5. 如果用户行为操作是需要特别严格的上下文支持，比如进行一个长时间的研究项目，那么建议关闭。 
## 最大token使用数
- **定义**: 开启token节约模式后，系统最大token使用数，超过次数后，智能体会自动保存记忆和token使用记录并且总结一个记忆总结文档(比对话记录消耗更少的token)给智能体作为上下文，避免智能体忘记之前的干了什么。
- **默认值**: 500000    
### 不同的需求下的最大token使用数
1. 如果用户行为操作是短期临时任务，比如查询新闻、查询天气等，那么建议最大token使用数为500000。
2. 如果用户行为操作是需要长期任务但是对上下文的要求不需要特别高，比如进行开发一个项目，那么建议最大token使用数为1000000。
3. 如果用户行为操作是需要特别严格的上下文支持但是又希望优化成本，比如进行一个长时间的研究项目，那么建议最大token使用数为2000000。
