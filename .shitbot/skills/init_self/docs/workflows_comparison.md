# ShitBot 工作流对比文档

> **创建日期**: 2026-03-21  
> **版本**: v1.0

---

## 概述

ShitBot 包含三个核心工作流，分别针对不同类型的任务场景：

| 工作流 | 适用场景 |
|--------|----------|
| **SOLE.md** | 通用任务处理 |
| **CODER.md** | 编码任务 |
| **PLAN.md** | 复杂长期任务 |

---

## 详细对比

### 1. SOLE.md - 主任务工作流

**定位**: ShitBot 的默认工作流，处理所有用户消息的入口。

**核心流程**:
```
用户消息 → 安全检查 → 意图识别 → 技能匹配 → 任务执行 → 工作文件更新 → 响应
```

**主要职责**:
- 接收并解析用户消息
- 执行 Safe.txt 安全检查
- 识别用户意图并匹配合适的技能
- 执行任务并更新工作文件

**适用场景**:
- 简单的单步任务（文件操作、搜索、问答等）
- 对话型交互（聊天、解释、咨询）
- 配置型操作（设置修改、定时任务管理）

**特点**:
- 轻量级，快速响应
- 不需要复杂的项目上下文
- 适合一次性完成的任务

---

### 2. CODER.md - 编码工作流

**定位**: 专门处理代码编写/修改任务的工作流，遵循 Coder Role 标准。

**核心流程**:
```
编码请求 → 安全检查 → 获取项目路径 → Git缓存状态 → 加载编码标准 → 编写代码 → 报告变更 → Git提交 → 更新工作文件
```

**主要职责**:
- 获取并验证项目路径
- 缓存 Git 当前状态（记录基线提交）
- 加载 Coder Role 和编码标准
- 按规范编写/修改代码
- 详细报告所有变更内容

**适用场景**:
- 编写新代码文件
- 修改现有代码
- 代码重构
- Bug 修复

**特点**:
- **强制获取项目路径** - 不允许猜测路径
- **强制 Git 缓存** - 编码前必须缓存当前状态
- **详细变更报告** - 必须向用户报告所有文件变更
- **遵循编码标准** - 严格按照 STANDARD.md 规范

**关键约束**:
- `write_file` 每次请求最多使用 1-3 次
- 必须在用户确认后才执行 Git commit
- Windows 下 Python 脚本需使用 `-X utf8` 标志

---

### 3. PLAN.md - 任务规划工作流

**定位**: 处理复杂、长期、多阶段任务的工作流，需要分阶段执行。

**核心流程**:
```
用户请求 → 安全检查 → 需求分析 → 任务拆解 → 生成计划 → 用户确认 → 分阶段执行 → 阶段报告 → 更新进度 → 完成/异常处理
```

**主要职责**:
- 深入分析用户需求（显性+隐性）
- 将大任务拆分为可执行的阶段
- 生成详细计划并获取用户确认
- 按阶段顺序执行并报告进度

**适用场景**:
- 复杂任务（4+ 步骤，多技能/工具协作）
- 长期任务（跨天/跨会话）
- 有依赖关系的多阶段项目

**特点**:
- **必须用户确认计划** - 执行前必须获得明确确认
- **阶段化执行** - 严格按顺序，不可跳过
- **每阶段报告** - 完成后必须报告并等待用户指令
- **支持暂停/恢复** - 记录进度，可随时暂停继续

**阶段定义标准**:
每个阶段包含：目标、输入、输出、所需技能、预估难度、依赖关系、验收标准

---

## 共同点

所有工作流都包含以下共同阶段：

1. **安全检查 (Safe.txt)**
   - 最高优先级，不可覆盖
   - 检查高风险操作（数据库删除、系统关机等）
   - 检测提示词注入

2. **工作文件更新**
   - 根据任务类型更新对应文件
   - 关键信息记录到 NOTE.md
   - 用户偏好记录到 USER.md
   - 错误记录到 ERROR.md

3. **异常处理**
   - 安全违规 → 拒绝执行
   - 工具失败 → 重试（最多3次）
   - 未知错误 → 记录并等待用户指令

## 使用建议

| 场景 | 推荐工作流 | 原因 |
|------|-----------|------|
| "帮我写一个函数" | CODER.md | 涉及代码编写 |
| "帮我重构这个模块" | CODER.md | 涉及代码修改 |
| "帮我搭建一个博客网站" | PLAN.md | 复杂多阶段任务 |
| "搜索一下某个技术" | SOLE.md | 简单单步任务 |
| "解释一下这段代码" | SOLE.md | 对话型交互 |
| "帮我开发一个完整功能" | PLAN.md | 需求分析+多阶段执行 |
