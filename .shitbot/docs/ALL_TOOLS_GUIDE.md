# ShitBot 完整工具使用指南 -- 该文档详细介绍了 ShitBot 中所有可用工具的使用方法、参数说明以及最佳实践

## 工具概述

ShitBot 提供了丰富的工具集，涵盖网络搜索、文件操作、定时任务、邮件发送、记忆管理、代码执行等多个功能模块。本文档将逐一介绍每个工具的使用方法。

---

## 网络搜索模块

### search_web - 网络搜索

#### 工具介绍
在网络上搜索信息，支持自定义搜索关键词和返回结果数量。

#### 工具参数
- `query` (必需): 搜索查询词，字符串类型
- `count` (可选): 返回结果数量，整数类型，默认值为 5

#### 使用示例
```
搜索最新的AI新闻，返回10条结果
参数：query="AI最新新闻", count=10
```

#### 最佳实践
- 使用具体的关键词可以获得更精准的搜索结果
- 如果需要更多结果，可以适当增加 count 参数值
- 建议配合 webbot_task 工具使用，获取详细内容

---

### webbot_task - 浏览器任务执行

#### 工具介绍
让WebBot执行浏览器操作任务，包括查看网页信息、点击网页、填写表单等浏览器修改功能。

#### 工具参数
- `query` (必需): 告诉WebBot要执行的任务，字符串类型

#### 使用示例
```
打开百度并搜索"人工智能"
参数：query="打开百度并搜索人工智能"
```

#### 最佳实践
- query 参数要明确描述需要执行的任务
- 支持复杂的浏览器操作流程
- 可以用于自动化网页交互

---

## 文件操作模块

### write_file - 写入文件

#### 工具介绍
给指定文件写入内容，如果文件不存在会自动创建。

#### 工具参数
- `file_path` (必需): 要写入的文件路径，字符串类型
- `content` (必需): 要写入的内容，字符串类型

#### 使用示例
```
在 D 盘创建一个文本文件并写入内容
参数：file_path="D:\\test.txt", content="Hello World"
```

#### 最佳实践
- Windows 路径需要使用双反斜杠 `\\` 或正斜杠 `/`
- 写入前确认路径存在，否则可能失败
- 大文件写入建议分批进行
- **注意**: 每次用户请求最多只能写入1-3个文件

---

### read_file - 读取文件

#### 工具介绍
读取指定文件的内容。

#### 工具参数
- `file_path` (必需): 要读取的文件路径，字符串类型

#### 使用示例
```
读取 D 盘下的配置文件
参数：file_path="D:\\config.json"
```

#### 最佳实践
- 确保文件路径正确且文件存在
- 大文件读取可能需要较长时间
- 注意文件编码格式

---

### append_to_file - 追加文件内容

#### 工具介绍
在指定文件末尾追加文本内容。

#### 工具参数
- `file_path` (必需): 要追加内容的文件路径，字符串类型
- `content` (必需): 要追加的文本内容，字符串类型

#### 使用示例
```
在日志文件末尾追加新记录
参数：file_path="D:\\log.txt", content="\\n2026-03-15: 新的日志记录"
```

#### 最佳实践
- 不会覆盖原有内容，只在末尾追加
- 适合日志记录、数据追加等场景
- 建议在追加内容前添加换行符 `\\n`

---

### copy_file - 复制文件

#### 工具介绍
复制指定文件到目标路径。

#### 工具参数
- `source_path` (必需): 要复制的文件路径，字符串类型
- `dest_path` (必需): 要复制到的目标路径，字符串类型

#### 使用示例
```
将文件从源目录复制到目标目录
参数：source_path="D:\\source\\file.txt", dest_path="D:\\backup\\file.txt"
```

#### 最佳实践
- 确保目标目录存在
- 如果目标文件已存在，会被覆盖
- 复制大文件时请耐心等待

---

### move_file - 移动文件

#### 工具介绍
移动指定文件到目标路径。

#### 工具参数
- `source_path` (必需): 要移动的文件路径，字符串类型
- `dest_path` (必需): 要移动到的目标路径，字符串类型

#### 使用示例
```
将文件移动到归档目录
参数：source_path="D:\\temp\\file.txt", dest_path="D:\\archive\\file.txt"
```

#### 最佳实践
- 移动操作会删除原位置的文件
- 确保目标目录存在
- 跨磁盘移动可能需要复制+删除两步操作

删除指定的文件。

## 定时任务模块

### once_after - 延迟执行一次

#### 工具介绍
在指定时间后执行一次任务。

#### 工具参数
- `time` (必需): 延迟时间（秒），整数类型
- `task` (必需): 任务描述，字符串类型

#### 使用示例
```
5分钟后提醒喝水
参数：time=300, task="提醒喝水"
```

#### 最佳实践
- 时间单位为秒
- 适用于一次性提醒任务
- 任务描述要清晰明确

---

### interval - 周期性执行

#### 工具介绍
按照指定时间间隔周期性执行任务。

#### 工具参数
- `interval_seconds` (必需): 时间间隔（秒），整数类型
- `interval_count` (必需): 触发次数，-1 表示无限次，整数类型
- `task` (必需): 任务描述，字符串类型

#### 使用示例
```
每小时检查一次系统状态，执行24次
参数：interval_seconds=3600, interval_count=24, task="检查系统状态"
```

#### 最佳实践
- interval_count=-1 表示无限循环
- 注意任务执行时间不要超过间隔时间
- 长期运行的任务要考虑资源占用

---

### daily_at - 每日定时执行

#### 工具介绍
每天在指定时间执行任务。

#### 工具参数
- `hour` (必需): 小时（0-23），整数类型
- `minute` (可选): 分钟（0-59），整数类型
- `task` (必需): 任务描述，字符串类型

#### 使用示例
```
每天早上8点发送日报
参数：hour=8, minute=0, task="发送日报"
```

#### 最佳实践
- 使用24小时制
- 时间格式要准确
- 适合日常固定任务

---

### cancel_timer - 取消定时任务

#### 工具介绍
取消指定的定时任务。

#### 工具参数
- `task_id` (必需): 任务ID，字符串类型

#### 使用示例
```
取消ID为 task_123 的任务
参数：task_id="task_123"
```

#### 最佳实践
- 可以使用 list 工具查看所有任务ID
- 取消后任务将不再执行
- 已执行的任务无法取消

---

### pause_timer - 暂停定时任务

#### 工具介绍
暂停指定的定时任务。

#### 工具参数
- `task_id` (必需): 任务ID，字符串类型

#### 使用示例
```
暂停ID为 task_456 的任务
参数：task_id="task_456"
```

#### 最佳实践
- 暂停后可以恢复执行
- 暂停期间任务不会触发
- 使用 resume_timer 工具恢复任务

---

### resume_timer - 恢复定时任务

#### 工具介绍
恢复已暂停的定时任务。

#### 工具参数
- `task_id` (必需): 任务ID，字符串类型

#### 使用示例
```
恢复ID为 task_789 的任务
参数：task_id="task_789"
```

#### 最佳实践
- 只能恢复已暂停的任务
- 恢复后任务按原计划继续执行
- 配合 pause_timer 工具使用

## 子智能体模块

### create_subagent - 创建子智能体

#### 工具介绍
创建一个新的子智能体（SubAgent），分配特定角色。

#### 工具参数
- `role` (必需): 子智能体的角色，字符串类型，例如"Frontend Developer"
- `description` (必需): 子智能体的职责描述，字符串类型

#### 使用示例
```
创建一个前端开发子智能体
参数：role="Frontend Developer", description="负责编写前端HTML/CSS/JavaScript代码"
```

#### 最佳实践
- 为每个专业任务创建对应角色的子智能体
- 角色描述要清晰明确
- 充分利用子智能体并行执行多个任务
- 子智能体不能递归创建子智能体，这是安全限制

---

### subagent_task - 分配任务给子智能体

#### 工具介绍
将任务分配给已创建的子智能体异步执行。

#### 工具参数
- `task` (必需): 任务描述，字符串类型
- `role_id` (必需): 子智能体的ID，从create_subagent或get_subagent获取

#### 使用示例
```
让前端子智能体实现登录页面
参数：task="实现一个响应式登录页面，包含用户名和密码输入框以及登录按钮", role_id="TASK_20240406_123456_001"
```

#### 最佳实践
- 任务描述要具体清晰
- 等待子智能体在后台完成任务，不会阻塞主对话
- 可以同时分配多个任务给多个子智能体实现并行处理
- 主智能体后续可以获取任务执行结果

---

### get_subagent - 获取所有子智能体状态

#### 工具介绍
列出所有已创建的子智能体及其当前状态。

#### 工具参数
无

#### 使用示例

#### 最佳实践

- 可以获取已完成任务的结果
- 了解各个子智能体的当前状态（空闲/忙碌/已完成）

## 注意事项

1. **路径格式**: Windows 系统路径使用双反斜杠 `\\` 或正斜杠 `/`
2. **参数类型**: 注意参数的数据类型，字符串需要加引号
3. **错误处理**: 工具执行失败时会返回错误信息，请根据提示调整
4. **安全性**: 执行删除、移动等操作前请确认路径正确
5. **性能**: 大文件操作和复杂命令可能需要较长时间，请耐心等待
6. **文件写入限制**: 每次用户请求最多只能写入1-3个文件
7. **删除确认**: 删除文件前必须向用户确认，定时任务中的删除操作将被拒绝
